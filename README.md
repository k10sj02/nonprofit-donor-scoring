# Donor Retention Propensity Framework

Lightweight donor lifecycle and propensity analysis focused on retention and recurring behavior in nonprofit fundraising.

---

## Overview

This project combines exploratory data analysis with a machine learning propensity model to help nonprofit fundraising teams identify which donors are most likely to give again. It includes a Jupyter notebook for data preparation and analysis, and a Streamlit app for interactive scoring and segmentation.

---

## Features

- Merges donation transaction records with donor profile data
- Engineers RFM-style features: recency, frequency, and monetary value
- Trains a logistic regression model to predict donor retention
- Scores and segments donors into Low / Medium / High propensity tiers
- Evaluates model quality with AUC-ROC
- Streamlit app with live filtering, segment performance table, and CSV export

## Quickstart

### Prerequisites

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) for dependency management

### Install dependencies

```bash
uv sync
```

### Run the notebook

Open `analysis.ipynb` in Jupyter to run the full EDA and data preparation pipeline. The final cell saves `outputs/donations_clean.csv`, which the Streamlit app loads automatically.

```bash
uv run jupyter notebook analysis.ipynb
```

### Launch the Streamlit app

```bash
uv run streamlit run app.py
```

The app will auto-load `outputs/donations_clean.csv` if it exists. You can also upload a CSV directly in the browser.

## Input Data Format

The app expects a CSV with at minimum these columns:

| Column | Type | Description |
|---|---|---|
| `donation_id` | string | Unique identifier per transaction |
| `donor_id` | string | Unique identifier per donor |
| `donation_date` | date | Date of the donation (YYYY-MM-DD) |
| `donation_amount` | float | Donation value in local currency |

---

## How the Model Works

1. **Train/future split** — donations are split at the 80th percentile by date. The earlier 80% is used to build features; the later 20% is used to define the target (did the donor give again?).
2. **Feature engineering** — per-donor aggregates are computed from the training window: donation count, total donated, average donation, and recency in days since last gift.
3. **Model** — a logistic regression classifier is trained on these four features to predict `donated_again`.
4. **Scoring** — all donors are scored with `predict_proba`, then segmented into Low / Medium / High tiers using equal-frequency binning (`pd.qcut`).
5. **Evaluation** — model quality is reported as AUC-ROC on the training cohort.

## Streamlit App

The app provides:

- **KPI metrics** — total donors, percentage in the High segment, and model AUC
- **Segment filter** — filter the table by Low / Medium / High propensity
- **Top Donors table** — sortable donor-level scores and features
- **Segment Performance table** — average retention rate per segment
- **Download button** — export the filtered results as a CSV

## Deployment

The app is deployable to [Streamlit Community Cloud](https://share.streamlit.io) for free.

1. Push the repo to GitHub (including `data/donations_clean.csv`)
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your GitHub repo
3. Set the main file to `app.py` and deploy

To update the dataset after deployment, push a new `data/donations_clean.csv` to GitHub. Streamlit Cloud will redeploy automatically within ~30 seconds.

Generate a `requirements.txt` for Streamlit Cloud compatibility:

```bash
uv pip compile pyproject.toml -o requirements.txt
```
